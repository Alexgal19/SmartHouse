## Pattern: Minimal Single Column

- **Conversion focus**: Single CTA per view
- **Typography**: Large headings, generous whitespace
- **Navigation**: Minimal clutter; bottom bar on mobile, sidebar on desktop
- **Sections per screen**: Hero headline, short description, benefit bullets (max 3), CTA, footer

| `--color-destructive` | `#DC2626` | Errors, delete, danger | `red-600` |
| `--color-ring` | `#2563EB` | Focus rings | `blue-600` |

### Semantic Usage Rules
- **Primary** = CTA buttons, active nav states, links
- **Accent (green)** = Success toasts, positive badges, "zakwaterowana" status
- **Destructive** = Delete, error states, failed interview
- **Background/Foreground** = Light mode base; dark mode inverted via CSS variables
- **Never use raw hex** in components — always use Tailwind semantic tokens (`bg-primary`, `text-foreground`, etc.)

### Dark Mode
- Use desaturated / lighter tonal variants, not inverted colors
- Test contrast independently per theme
- Tokens already defined in `globals.css` `.dark`

## Anti-Patterns (Avoid)

| Anti-Pattern | Why | Fix |
|--------------|-----|-----|
| Cluttered interface | Reduces task focus | Single CTA, whitespace, progressive disclosure |
| No visual presence / flat everything | Looks unfinished | Subtle shadows, elevation, depth |
| Emoji as icons | Font-dependent, inconsistent | Lucide SVG icons only |
| Hardcoded hex colors | Breaks theming, dark mode | Use Tailwind semantic tokens |
| Placeholder-only inputs | Accessibility failure | Always pair with visible `<label>` |
| Instant state changes (0ms) | No perceived feedback | 150–300ms transitions |
| Layout-shifting press states | Jittery interactions | Use color/opacity/elevation, not width/height |

## Forms & Feedback

### Labels
- Every input must have a visible `<label>` or `aria-label`
- Use `htmlFor` + `id` association
- Placeholder = hint only, never the sole label

### Error Handling
- Show errors near the field (below input)
- Use `FormMessage` from shadcn/ui
- Toast for global errors only

### Loading States
- Disable submit button during async
- Show spinner inside button (`animate-spin`)
- Skeleton screens for initial data load (>300ms)

### Toast Rules
- Auto-dismiss: 5 seconds (already fixed)
- `aria-live="polite"` on toast viewport (already applied)
- Limit: 1 toast at a time (`TOAST_LIMIT = 1`)
- Variant: `destructive` for errors, default for success

- Bottom nav: max 5 items (compliant)
- Icon + label for every item

- Sheet for overflow actions

### Desktop

## Pre-Delivery Checklist (Per Component / Page)

### Visual Quality
- [ ] No emojis used as icons
- [ ] All icons from Lucide, consistent stroke
- [ ] Semantic theme tokens (no raw hex in components)
- [ ] Pressed states without layout shift

### Interaction
- [ ] Tap feedback within 80–150ms
- [ ] Touch targets ≥44px
- [ ] Disabled states visually clear
- [ ] Focus rings visible (2px `ring-primary`)

### Accessibility
- [ ] Labels for all inputs
- [ ] `aria-label` for icon-only buttons
- [ ] Color is not the only indicator (add icon/text)
- [ ] Keyboard navigation supported
- [ ] `prefers-reduced-motion` respected

### Responsive
- [ ] Tested on 375px width
- [ ] No horizontal scroll
- [ ] Bottom padding for fixed nav (`pb-20` or similar)
- [ ] Safe areas respected
